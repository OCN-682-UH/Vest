Why This Plot Is Bad

  • Too much chaos

    • Uses all launches and ISO codes, which means some big groups like the European Space Agency are missing.
    • Points are everywhere—Europe, USA, and Asia are so packed it’s basically impossible to tell what’s going on.

  • Misleading visuals

    • Point size exaggerates some launches and hides smaller ones.
    • Pink background + orange land + purple borders = headache.
    • Map projection distorts places, making it harder to interpret.

  • Bad labels and legends

    • Axes are “Longitude (kinda)” and “Latitude (ish)”—very helpful (not)
    • Title/subtitle colors fight with each other
    • Legend is messy, hard to read, and doesn’t really tell you anything.

  • Animation doesn’t help

    • Points move over time, but it doesn’t clarify anything. Instead, it just adds to the noise and makes it hard to see the patterns.

  • Overall effect

    • This plot breaks basically every rule from Healy and Wilke about making data readable.
    • It’s fun to look at for a second, but you learn nothing from it.

Data prep

Click me for code! 
# Select top 5 entities by total launches (exclude "World")
top_5_entities <- space_objects %>%
  filter(Entity != "World") %>%
  group_by(Entity) %>%
  summarise(total_launches = sum(num_objects, na.rm = TRUE), .groups = 'drop') %>%
  arrange(desc(total_launches)) %>%
  slice_head(n = 5) %>%
  pull(Entity)

# Prepare data for plotting
top_data <- space_objects %>%
  filter(Entity %in% top_5_entities) %>%
  group_by(Year, Entity) %>%
  summarise(launches = sum(num_objects, na.rm = TRUE), .groups = 'drop') %>%
  complete(Year = 1960:2020, Entity, fill = list(launches = 0))

# Reorder entities: largest contributer to the back of the stack
entity_order <- top_data %>%
     group_by(Entity) %>%
     summarise(total = sum(launches)) %>%
     arrange(desc(total)) %>%
     pull(Entity)
top_data$Entity <- factor(top_data$Entity, levels = entity_order)

goodplot <- ggplot(top_data, aes(x = Year, y = launches, fill = Entity)) +
     geom_area(alpha = 0.85, color = "white", size = 0.25) +
     scale_fill_viridis_d(option = "mako", end = 0.95) +
     scale_y_sqrt(
         expand = expansion(mult = c(0, 0.05))
     ) +
     scale_x_continuous(
         breaks = seq(1960, 2020, by = 10),
         limits = c(1960, 2020),
         expand = expansion(mult = c(0, 0))
     ) +
     labs(
         title = "The Space Launch Race: 1960–2020",
         subtitle = "Launches by the top 10 spacefaring entities",
         x = "Year",
         y = "Number of Launches (sqrt scale)",
         fill = "Entity"
     ) +
     theme_minimal(base_size = 14) +
     theme(
         plot.title = element_text(face = "bold", size = 20, hjust = 0.5),

         axis.text = element_text(color = "gray20", face = "bold"),
         axis.title = element_text(face = "bold", size = 13),
         legend.position = "bottom",
         legend.justification = "center",
         legend.title = element_text(face = "bold", size = 12),
         legend.text = element_text(size = 11),

     )

Why this plot is so good

  • Focus on the important stuff

    • Only shows the top 5 countries, so it’s easy to see who actually dominates space launches.
    • Stacking makes sense: biggest contributor (USA) is at the back, smaller ones on top.

  • Colors and scaling that actually help

    • Y-axis uses a square root scale, so small countries aren’t completely invisible.
    • Viridis “mako” palette looks nice and is easy to tell apart—even if you’re colorblind.

  • Clear labels and axes

    • X-axis shows decades, so you can see trends over time.
    • Axis titles are bold and readable.
    • Legend is simple.

  • Clean theme

    • theme_minimal() keeps the gridlines and background subtle.
    • No distracting chart junk—just the data.
